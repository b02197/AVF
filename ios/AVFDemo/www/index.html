    Week 1:

    Less obvious perks of using 'live' data resources include logins and payments. How could Twitter be used in these two scenarios?

    Twitter could help with a secure login for the users. It would also be easy way for users to register with the app. For payments the user would have a secure way to make payment transaction without having to worry about the app being about to grab the users personal information.

    Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

    JSON with padding will allow us to make a remote request from a different domain server to fetch data for your app.

    Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

    Something like Audiosocket MaaS APi would allow you to steam and search for music for a media player in your app. MapQuest Directions API would use your GPS function.

    Please describe any design and functionality intentions you may have for your Demo app components this month.

    I would like to use the map and GPS together to make a map the will have a preprogrammed point for like a restaurant and give you your location and have a map with directions to take you there. I also intend to either use the accelerometer for a step counter or to make a character walk on a map.

    Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

    Some past lessons learn for me are not to put off a project till the last minute. Trying to rush a project in a few hours isn't fun but can be done. I learned that when I do a little research on the function that I would like to create, it is easier to implement it in the code with fewer errors when just trying to wing it for hours till you figure it out.
